Business Logic URL HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all the movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/title GET None A JSON object holding data about a single movie, containing a description, genre, director, image url, featured/not featured, ( description : "summary info", genre : "comedy", director : "Spielberg", image url : "imgur.com", featured : "featured" )
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/genre/[title] GET None A JSON object holding data about a single movie by genre by name/title ( genre : "drama", name/title : "Gone With The Wind")
Return data about a director (bio, birth year, death year) by name /movies/:title/director GET None A JSON object holding data about a director by name ( bio : "USA", birth year : "1992", death year : "-" )
Allow new users to register /users POST A JSON in the format ID, Username, Password, Email, and Birthday
Allow users to update their user info (username) /users/:Username PUT None A JSON in the format Username, Password, Email, all required in strings, plus Birthday with date
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) /users/:Username/Movies/:MovieID POST A text message indicating whether a movie has been added
Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later) /movies/:movieid DELETE None A text message indicating whether a movie was successfully removed
Allow existing users to deregister (showing only a text that a user email has been removed—more on this later) /users/:Username DELETE None A text message indicating whether a user was successfully removed
